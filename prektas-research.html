Pre-KTAS → 응급의료센터 등급 추천

구급대원이 현장에서 내린 5자 코드로, 갈 수 있는 병원의 등급을 정한다.

이 연구는 Pre-KTAS 4,689개 코드와 EMRIS 종합상황판의 27개 중증응급질환 Y코드 사이를, 0–3개의 추가 질문만으로 연결할 수 있는지 묻는다. 정답은 "Y코드 1개 확정"이 아니라 수용 가능한 응급의료센터 등급이다. 진단 확정을 현장에 떠넘기지 않으면서 자원 배분은 제안할 수 있다.

연구: Claude + Pandoll-AI Codebook v2.0.0 · Tier v1.1.0 Sample: 4689 Pre-KTAS codes × 27 Y-codes
01문제 정의

코드만으로는 수술·시술 적응을 결정할 수 없다.

Pre-KTAS는 구급대원이 현장에서 환자의 긴급도를 1–5로 평가하기 위한 triage 도구다. 반면 EMRIS 종합상황판의 27개 Y코드는 병원 측 수술·시술·자원 적응을 정의한다 (Y0010 심근경색 재관류중재술, Y0020 뇌경색 재관류, Y0120 중증화상 등). 둘은 같은 환자를 서로 다른 층위에서 본다.

이 프로젝트의 초기 가설 — "Pre-KTAS 코드를 0–3개 질문으로 Y코드 1개에 확정 매핑한다" — 은 임상적으로 부적절하다. 뇌경색과 뇌출혈은 CT 없이 현장 감별이 불가능하고, 흉통의 감별진단은 병원 도착 후 ECG·심근효소·echo로 한다. 구급대원에게 진단 확정 책임을 지우는 건 119 운영 철학에 어긋난다.

확정 매핑 대신 수용 가능한 자원 등급만 제시한다. 확정은 병원이 한다.
02프레이밍 전환

Y코드 확정 → 응급의료센터 등급 추천.

4,689 코드에 대해 권역/지역센터/지역기관 중 수용 가능한 tier 집합을 추천한다. "권역 전용"은 극소수이고, 대부분은 권역과 지역센터가 공동 커버하며, 경증은 지역기관에서 충분하다.

CSV 정본 → JSON 스키마 강제 → rule-based 매핑 → tier 룩업.

       │                                              │
       ▼  generate-prektas-codebook.mjs                │

       │                                              │
       ▼  build-prektas-to-y-mapping.mjs               │
research/prektas-to-y-mapping.json ◄──────────────────┘
       │         (12 domain rules + 13 question catalog)
       │
       ▼  build-prektas-tier-recommendation.mjs
research/prektas-tier-recommendation.json
       │         (Y후보 교집합 + grade fallback)

중간 산출물은 모두 재현 가능하다. 정본 CSV의 SHA-256이 schema에 박혀 있고, ajv가 level2.code ↔ code[1] 같은 구조적 불변식을 강제한다. 이전 phase의 KTAS 오염 사고(§05)에서 얻은 교훈이다.

04규칙

Y-tier는 교집합, grade는 fallback.

Pre-KTAS 엔트리에 매칭된 Y코드 후보가 있으면 각 후보의 수용 가능 tier 리스트를 교집합해 최종 acceptable을 결정한다. 하나라도 권역 전용 Y코드가 섞이면 전체가 권역 전용이 된다 (안전 보수적). Y후보가 없으면 grade에 따라 fallback.

조건acceptable tiers (우선순위)전략
Y후보 있음, 교집합 = [regional]권역regional_only
Y후보 있음, 교집합 = [regional, local_center]권역 → 지역센터regional_or_local_center
Y후보 없음, grade 1–2권역 → 지역센터regional_or_local_center
Y후보 없음, grade 3지역센터 → 지역기관local_center_preferred
Y후보 없음, grade 4–5지역기관 → 지역센터local_institution_preferred

연령 분기는 질문 없이 해결된다. Pre-KTAS 코드 첫 문자(C=성인, D=소아)가 이미 답하고 있으므로 Y0081 vs Y0082(위장관 내시경) 같은 연령 짝은 추가 질문 0개.

05이전 실수의 교훈

정본이 아닌 데이터로 만들어진 연구는 조용히 틀린다.

이 프로젝트의 초기 phase에서 Pre-KTAS 코드북으로 쓴 데이터의 91.7%가 KTAS 용어로 오염되어 있었다. 원인: "Pre-KTAS 코드로 KTAS 이름을 역인덱싱한 가공물"을 정본으로 오인한 것. level2 카테고리 이름이 전부 KTAS 쪽(산부인과·비뇨기과·이비인후과 등)이었고, 진짜 Pre-KTAS 분류(임신/여성생식계·비뇨기계/남성생식계·입·목/얼굴 등)로 교체하는 데 상당한 데이터 복구가 필요했다.

구조적 교훈

validator 스키마를 아무리 정교하게 짜도 입력의 출처가 틀리면 전부 무의미하다. 첫 질문은 언제나 "이 파일이 정말 우리가 말하는 그 파일인가?"여야 한다. 현재 파이프라인은 정본 CSV의 SHA-256을 기록하고, generator는 CSV 헤더와 코드 구조를 양쪽에서 검증한다.

06한계

무엇을 아직 모르는가.

False negative rate는 미측정이다. 이 전략이 "중증 환자를 지역기관으로 잘못 보낸" 비율을 수치화하지 않았다. 연구 프로젝트의 동반 데이터셋 source-prektas.csv에 225,018건의 실측 응급실 방문이 있고, 각 케이스는 병원 도착 후의 최초KTAS분류과정과 퇴실진단코드를 담고 있다. 이를 뒤집어 본 전략의 tier 추천이 실측 진단과 얼마나 맞는지 측정하는 것이 다음 phase의 1순위 과제다.

두 번째는 지역별 편차다. 현재 Y-tier 분류는 전국 평균이며, 지방 일부 권역에는 수부접합이 없고 수도권 일부 지역센터에는 NICU가 있는 등 현실 편차를 반영하지 않는다. EMRIS 실시간 병상 데이터(emris-data/devdocs/*.json)와 결합 시 구체화 가능하다.

세 번째는 임상 리뷰의 부재다. Y코드 tier 분류 1차안은 응급의학 전문의 검토 전이다. 특히 Y0131/Y0132(수부·사지접합)와 Y0120(중증화상) 분류는 현실 운영과 대조 필요.

07재현

한 번의 명령으로 전체 파이프라인을 재실행할 수 있다.

git clone https://github.com/Pandoll-AI/emris-119chat
cd emris-119chat
npm install

npm run codebook:rebuild               # 정본 CSV → JSON + 스키마 검증
npm run research:all                   # Y매핑 v0.1 + tier 추천 v1.1
npm run build:html                     # standalone HTML 두 개 재생성

open prektas-hospital-recommender.html # 추천 도구
open prektas-research.html             # 본 문서

각 단계의 산출물은 정본 데이터의 SHA-256을 포함하므로 원본 CSV가 바뀌면 즉시 감지된다.
