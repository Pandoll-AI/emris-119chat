PROTOCOL PREKTAS-VAL-2026-001 · PREREGISTERED
진단정확도 검증 프로토콜 — STARD 2015 준수

Pre-KTAS + 0–3 질문은 정말 중증응급환자를 가려내는가?

본 프로젝트의 v0.1 룰 알고리즘이 EMRIS 27 중증응급질환을 식별하는 능력을, 광주 지역 ED 225,017건의 실측 방문 데이터와 사전 등록된 통계 분석 계획으로 측정한다. 정답은 "Y코드 1개 확정"이 아니라 이항 sensitivity·specificity다. 본 문서는 데이터 잠금 이전에 동결된 사전 분석 계획이다.

Protocol v1.0 Frozen: 2026-04-25 Codebook v2.0.0 Sample N: 225,017 Source: Pandoll-AI / EMRIS 119
01배경

Coverage는 정확도가 아니다.

Phase 3에서 Pre-KTAS → EMRIS 27 Y코드 매핑 v0.1 룰을 만들었고, 4,689개 코드 중 NaN%가 0–3 질문 안에 매핑되었다. 이것은 진단정확도가 아니다. 단순히 "알고리즘이 답을 낸 비율"이다.

응급의료에서 의미 있는 질문은 다르다. 우리 알고리즘이 실제 중증응급환자(EMRIS 27 Y코드 해당)를 정확히 식별하는가? 비중증 환자를 정확히 거르는가? Under-triage는 환자를 위험에 빠뜨리고 over-triage는 권역응급의료센터 자원을 낭비한다. 둘 다 측정 없이는 알고리즘 배포가 불가능하다.

사전 등록된 분석 계획 + 동결된 reference standard + 동결된 index test. 결과가 어떻게 나오든 보고한다.

본 연구의 위치

  • Retrospective observational diagnostic accuracy study
  • STARD 2015 가이드라인 준수
  • 알고리즘 임상 인증이 아니다 — baseline 측정과 v0.2 우선순위 도출이 목적
  • 본 protocol은 데이터 잠금 이전에 commit·동결된다 (preregistration)
02가설

측정 가능한 임계값으로 정의한다.

임계값(0.85·0.90·0.03·0.70)은 응급 분류 도구 평가의 통상 수치를 참고했고, 응급의학 전문의 자문에서 변경될 수 있다. 변경 시 본 protocol에 amendment로 기록한다.

H1
Index test (≤3 질문)의 sensitivity ≥ 0.85
primary · 95% lower CI ≥ 0.85 → 통과
H2
Index test (≤3 질문)의 specificity ≥ 0.90
primary · 95% lower CI ≥ 0.90 → 통과
H3
추가 질문 1개당 sensitivity 한계 증분 ≥ 0.03 (3 질문까지 누적)
secondary · paired McNemar p<0.05 + 효과크기 ≥0.03
H4
tier 권고와 실제 이송 종별 일치율 ≥ 0.70
secondary · 95% lower CI ≥ 0.70

결정 규칙

  • 모두 통과 → v0.1 freeze + prospective external validation phase 9 진입
  • H1 미통과 (sensitivity 부족) → 임상 활용 불가, v0.2 우선순위: false negative top 100 audit + rule gap fill
  • H2 미통과 (specificity 부족) → over-triage, v0.2 우선순위: 카테고리 over-inclusion 제거 (눈/정신/산부인과 pre-filter)
  • 모든 케이스: under-triage 절대값 보고 (윤리적 우선)

04데이터 정합성

분석 시작 전 정렬해야 할 두 가지.

⚠ Phase 8a 우선 과제 — 코드 체계 정렬

1차 스캔 결과 본 프로젝트의 정본 codebook은 5자 형식 (예: CAAAA, group prefix C/D)이지만, 실측 CSV의 최초KTAS분류과정 컬럼은 6자 형식 (예: AIACA0)이다. 첫 글자가 "A"인 케이스가 175,237/225,017 = 77.9%를 차지한다. 두 코드 체계의 정렬 매핑(crosswalk) 작성이 분석 전제 조건이다.

05분석 계획

사전 등록된 통계 분석.

Primary analysis (H1·H2)

이항 변환: predicted Y_candidates ∩ ground truth ≠ ∅로 TP 결정.

  • Sensitivity = TP / (TP + FN)
  • Specificity = TN / (TN + FP)
  • PPV / NPV / F1 / Cohen's κ / Balanced accuracy
  • 95% Wilson CI for proportions

Secondary analyses

  • 다항 confusion matrix: 27 Y + 1 unmapped = 28-class. per-class precision·recall·F1, macro-F1, Top-3 accuracy.
  • 한계효용 (H3): 0/1/2/3 질문 시나리오 4종 sensitivity, paired McNemar's test.
  • 운영 정확도 (H4): tier_strategy의 acceptable에 실제 종별 포함 비율.

Stratified analyses

  • 지역 (17개 시도)
  • 연령군 (소아 <18, 성인 18–64, 노인 ≥65)
  • Pre-KTAS grade (1–5)
  • 의식상태 (AVPU)
  • 종별 (권역/지역센터/지역기관)
  • 시간대 (주간/야간), 요일 (평일/주말)

Sensitivity analyses

  • ICD-10 cluster의 inclusive vs strict 정의 둘 모두 보고
  • 결측: complete case vs multiple imputation
  • 다중 ICD-10 라벨: first-listed vs any-match
  • 5자/6자 crosswalk 실패 코드 exclude vs lenient
  • Bootstrap 1000 (seed=20260425)

표본 크기 정당화

Sensitivity 0.85 ± 0.01 (CI 폭 ≤ 0.02) 추정에 필요한 true positive ≈ 4,900. 추정 prevalence 5% × N=225,017 → 예상 TP ≈ 11,250. 충분. Power analysis는 H0 검정이 아니므로 미수행.

09한계

사전에 인정한 한계.

  1. 단일 cohort, retrospective: prospective validation 부재. 외부 데이터 일반화는 향후 phase.
  2. ICD-10 reference standard imperfection: Y코드는 시술·자원 정의이지 진단 정의가 아니다. ICD-10 → Y cluster는 임상 자문 1인 기반 근사.
  3. Pre-KTAS 코드 부여의 inter-rater reliability는 본 연구 범위 외. 구급대원이 동일 환자에 동일 코드를 부여하는지는 별도 연구.
  4. 알고리즘 v0.1은 키워드 휴리스틱이며 임상 검증 전. 본 연구는 baseline 측정이며 알고리즘 임상 인증이 아니다.
  5. 산부인과·정신과·안과 카테고리는 알려진 over-inclusion (v0.1 §5-3). v0.2 우선 개선 대상.
  6. 추가 질문 답변 시뮬레이션은 best-case. 실제 구급 현장에서 답변 정확도는 별도 측정 필요.
  7. 시간 변동성: 2025–2026 데이터, 계절·코로나 후 변화 가능. period sub-analysis로 부분 완화.
